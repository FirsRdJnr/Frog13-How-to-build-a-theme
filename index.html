Optimization image sizes, using fonts, CSS gradients

To help FrogOS run effectively, we need to be careful what we put in there.

Avoid large background images

  • Slow load times
  • Problems switching between themes
  • Compression software, like kraken

Load times, iPad loading images - feedback on one forum suggest that you preload large .png files as this is where iPads fall down, but, unfortunately, we can’t preload images in themes.

File sizes

  • Combined files 800kb or lower
  • Need to be packaged up

All files, including images, in a theme folder should not exceed 800kb, we need to send these themes, sometimes over wifi to mobile devices, so the smaller you can make a theme, the better.

Embed custom fonts

  • Font Squirrel generator
  • Download and embed Google fonts
  • Header base sizes

Don't link to them from the web. Google fonts, or fonts you’ve bought can be turned into webfonts easily by using an online font generator like http://www.fontsquirrel.com/tools/webfont-generator. Different browsers require different font types to render the text. Remember, fonts have different base sizes, which means you must check that the overall size of text that uses the font are legible.

CSS over Images

  • CSS box shadows & more css3generator.com
  • CSS gradients Ultimate CSS Gradient Generator
  • Header base sizes

Use css gradients and box shadows instead of background images, CSS file sizes are vastly smaller than images and can achieve nicer results than using a low res image.

Structure .EJS, HTML, CSS, LESS, Navigation, Theme builder

A Theme's directory is system generated and is stored in FrogOS/internal/theme/theme folder/. The Theme is then called using JavaScript within FrogOS. In a theme directory there is no support for subfolders so all images etc. need to be in the root folder - this will be changing soon.

.ejs = embedded javascript

  • Combines data and a template to produce HTML
  • JavaScript between <%= %> adds HTML to the result.

An .ejs file compiles the HTML and CSS that makes up a theme into a readable file for FrogOS. It's still easier to build in HTML & CSS so you can view your work, then add both to an ejs file later.

Build in LESS for scoped CSS

  • Use LESS if you can.
  • Nested Rules
  • Crunch is a great Less compiler

.reveal {
    h1, h2, h3, h4, h5, h6 {
        color: #FFFFFF;
    }
    section {
        height: 800px;

        h2 {
            font-size: 5em;
            margin-bottom: 0 auto;
            text-align: center;

            span {
                color: #B3E0FD;
                display: block;
                font-size: 0.7em;
                line-height: 0.9em;
            }
        }
        &#lets-build {
            margin-top: 100px;
            h2 {
                font-size: 10em;
            }
        }
    }
}

Rather than constructing long selector names to specify inheritance, in Less you can simply nest selectors inside other selectors. This makes inheritance clear and style sheets shorter.

Navigation

  • Cheat!
  • No unique classes
  • Remember the + page

This is the trickiest part of any theme. As navigation much like the content and user generated, this means that each link has a unique ID generated by the system that we can’t predict in a theme. This means that you have to target levels of navigation by the UL class, data-menu-level="2" for example. You can also choose from two types of navigation, so you could have a nested navigation (where all the navigation is together), or you could choose to have 3 sets of individual navigation set out thorough the theme - there are no working examples of this in FrogOS as it will require a certain type of design, but this is the closest approximation we’d have to it being used. Userful to have the add page function in a different colour to normal.

Text colours

  • Widget colours

Keep in mind the content background colour when adding text styles there has been issues with white text on white backgrounds. Widgets have different colourd headers and can be overriden by accident.

No javascript

  • Frog OS is built with Javascript
  • Conflicts
  • Use classes not ids

As FrogOS’s interface is built using JS, we can’t allow custom scripts as we won't know how this will interact with existing JS on the page

Browser support

  • No more IE7!
  • All CSS2 & most CSS3 supported
  • Graceful degradation
  • Basically...avoid IE

JS was unstable in ie7 and as such, we moved away from this. This also means that most CSS2 styles will work with the platform, there are issues with CSS3 in ie8, but we encourage graceful degradation.
